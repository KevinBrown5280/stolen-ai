# As a pension participant, I can view Document History without seeing internally-excluded document types so that only documents intended for me are visible

## Decisions
- **Modify GetExcludedPaperDocumentTypes() directly to read cm_setting first, fall back to hard-coded list**: Keeps single responsibility (return exclusion list), minimal diff, caller unchanged
- **Setting key: SettingType='ESS_DOCUMENTS', SettingCode='EXCLUDED_PAPER_DOC_TYPES'**: Follows existing naming convention; descriptive and scoped
- **No admin path changes needed**: Admin code path never calls IsPaperDocument — exclusion logic is already scoped to participants
- **Inline comma-separated parsing (no utility method)**: 3-4 lines, no evidence of reuse elsewhere, avoids over-engineering
- **No automated tests — manual/integration verification**: No test project exists in the solution; creating one is out of scope for this story


## Tasks
### config-driven-exclusion: Add ICacheCmSettingDataService and modify GetExcludedPaperDocumentTypes to read cm_setting
In DocumentsRepository.cs: (1) Add ICacheCmSettingDataService IoC property following existing pattern (see NonEmployeeAccessRepository). (2) In GetExcludedPaperDocumentTypes(), call CacheCmSettingDataService.GetItem("ESS_DOCUMENTS", "EXCLUDED_PAPER_DOC_TYPES")?.Value. (3) If value is non-null and non-empty after Trim(): split on comma, trim each entry, filter empties, return as List<string>. (4) If null/empty: return existing hard-coded list (PEN_AFN, SUMMARY_PLAN_DESCRIPTION_401K, SUMMARY_PLAN_DESCRIPTION_BENEFIT_PLAN, SUMMARY_PLAN_DESCRIPTION_HW_REQ_ENROLL). Unknown codes are kept in the list — the Contains() check in IsPaperDocument simply won't match, which is safe.
